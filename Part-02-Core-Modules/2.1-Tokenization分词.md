# 1.1.1 分词定义

**定义**: **Tokenization**是将一个整体（例如词、短语、句子、段落甚至语音、图像）分割成较小单位（被称为token）的过程；中间使用的工具叫做**Tokenizer**

# 1.1.2 为什么需要进行分词(Why Need Tokenization Before LLM Time)

在大语言模型之前，自然语言处理（NLP）会有非常多的子任务，比如命名实体识别/词性标注等，分词是这些子任务的基础，分词准确度直接决定了后面的词性标注、句法分析、词向量以及文本分析的质量。

相比之下，**英文语句使用空格将单词进行分隔**，除了某些特定词，如New York等，大部分情况下不需要考虑分词问题。

但是，**中文则不同，因为其天然缺少分隔符，需要读者自行分词和断句**。故在做中文自然语言处理子任务时，需要先进行分词。中文分词难点主要集中在**分词标准，切分歧义和未登录词**三部分。

- **分词标准不统一**：由于缺乏明确的分词标准，中文分词在实际应用中面临一致性问题。
	-  例如，“中华人民共和国”在某些任务中会被当作一个整体词语，而在另一些任务中则被拆解为“中华”、“人民”、“共和国”三个词。不同机构、不同领域对“词”的定义不一致，造成训练数据和测试数据之间存在偏差， 影响模型泛化能力。
-  **切分歧义：** 组合型歧义、交集型歧义、真歧义。
	- **组合型歧义**：例如“研究生活动”可以切为“研究生活动”（正确）或“研究生活动”（错误），前者为名词短语，后者则失去了原意。
	- **交集型歧义**：例如“老人管理系统”可以切为“老人管理系统”（系统是给老人用的）或“老人管理系统”（系统是用于管理老人的），两种切法在不同上下文中可能都合理。
	- **真歧义**：例如“今天天气不错”，既可以切为“今天天气不错”，也可以切为“今天天气不错”（如果“今天天” 是专有名词，比如品牌名）。这种歧义无法通过规则轻易判断，需要结合上下文甚至知识库辅助判断。
- **未登录词处理**：新词涌现，中英文混杂
	- 也被称为“OOV（Out-of-Vocabulary）问题”，指的是词典中未收录的新词或罕见词，例如网络热词“内卷”、 “躺平”，或中英文混杂如“ChatGPT模型”。这类词通常无法通过词典识别，容易被误切或漏切，从而造成语义理解偏差。同时，随着新词不断出现，维护词典的成本也不断上升，传统分词系统难以应对。
